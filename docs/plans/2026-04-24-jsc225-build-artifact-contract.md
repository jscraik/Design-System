## Purpose

JSC-225 decides the tracked build-output and screenshot artifact contract that
JSC-223 intentionally deferred. The goal is to make the repository easier for
agents to reason about: source files, package manifests, docs, tests, and
workflow contracts are authority; local build products and ad hoc visual
captures are regenerated outputs.

## Decision

Do not track generated `dist/**` package/app outputs or Storybook screenshot
captures.

The canonical source of truth is:

- source files under `src/**`;
- package manifests and workspace scripts that declare public package entry
  points;
- release workflows that build before packing or publishing;
- committed Playwright visual baselines under `tests/visual/__snapshots__`;
- CI artifacts and Argos comparisons for generated visual evidence.

`dist/**`, app `dist/**`, and `platforms/web/apps/storybook/screenshots/**`
remain ignored by `.gitignore` and must be regenerated by the relevant build or
visual command when needed.


## Publish Contract

Package manifests may continue to expose `dist` paths in `main`, `types`,
`exports`, `bin`, and `files`. That is a package-output contract, not a
source-control contract.

Publishing and packing must build first:

- `.github/workflows/publish-astudio.yml` runs `pnpm build:astudio` before
  `pnpm publish:astudio`.
- `.github/workflows/release-guidance.yml` runs
  `pnpm -C packages/design-system-guidance build` before `npm pack --dry-run`
  and before `npm publish`.
- Local package checks that execute built CLIs should use the package build or
  package start script first, for example `pnpm -C packages/cli build` before
  `node packages/cli/dist/index.js ...`.

If a future package must publish checked-in generated files, it needs an
explicit exception in `.gitignore`, a regeneration command, and a validation
gate proving the checked-in artifact is fresh.

## Visual Artifact Contract

`platforms/web/apps/storybook/screenshots/**` is generated capture output, not
canonical review evidence.

Keep visual authority in:

- Playwright baselines under `tests/visual/__snapshots__`;
- Playwright reports and test artifacts produced during CI/local runs;
- Argos CI comparisons and baselines.

Manual screenshot directories may exist locally, but they should not be
committed unless a future workflow promotes a specific path as a documented
baseline source.

## Remaining Cleanup Surface

JSC-225 resolves the broad tracked build-output and screenshot decision. The
remaining tracked ignored files are separate cleanup categories and should stay
with their dedicated follow-up issues:

- generated source regeneration contract: JSC-226;
- stale reports and plan authority index: JSC-227;
- guard for newly tracked ignored runtime artifacts: JSC-228;
- orphaned scripts and prototype package lifecycle: JSC-229.
